## Funcionalidades/Hisória de Usuario

1. O sistema deve permitir o usuário manter o cadastro do mesmo, que possui os seguintes dados: e-mail, senha, nome, CPF, telefone, departamento.

1. O sistema deve permitir ao técnico manter o cadastro do mesmo, que possui os seguintes atributos: e-mail, senha, nome, CPF, telefone, departamento, especialidade.

1. O sistema deve permitir ao administrador manter um cadastro de serviços oferecidos, tendo nome, descrição do serviço, área (hardware ou software) e nível de prioridade.

1. O sistema deve permitir ao administrador manter um cadastro de categorias de serviços, onde serão divididos os serviços prestados, tendo uma descrição da categoria, área;

1. O sistema deve permitir ao usuário realizar chamados. Estes devem possuir a data de abertura, serviço a ser prestado, mensagem, prazo para solução, status (em análise, em aberto, em andamento, cancelado, concluído), data da solução.

1. O sistema deve ter uma função de acompanhamento do chamado, para que o técnico faça o atendimento e depois de solucionado o cliente avalie a solução e feche o chamado.
